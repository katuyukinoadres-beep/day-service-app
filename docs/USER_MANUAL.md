# Part A：職員編

**このパートは、現場で毎日記録を入力する職員さん向け**です。

## A-1. 初めてのログイン

### A-1-1. ログイン画面を開く

施設長から渡された URL（[https://day-service-app.vercel.app/](https://day-service-app.vercel.app/)）をスマホで開きます。

### A-1-2. メールアドレスとパスワードを入力

- **メールアドレス**: 施設長から伝えられた自分のメールアドレス
- **パスワード**: 仮パスワード（初回のみ）

「ログイン」をタップ。

### A-1-3. パスワードをすぐ変更

右下「設定」→「パスワード変更」で、自分しか知らないパスワードに変更してください。

### A-1-4. スマホのホーム画面に追加（おすすめ）

毎回ブラウザの URL を入力するのは面倒なので、**ホーム画面にアイコンを置く**と便利です。

**iPhone（Safari）の場合**:
1. 画面下の「共有ボタン」（□に↑）をタップ
2. 「ホーム画面に追加」を選択
3. 名前を確認して「追加」

**Android（Chrome）の場合**:
1. 画面右上の「⋮」メニュー
2. 「ホーム画面に追加」
3. 「追加」

次からは **通常のアプリと同じようにアイコンをタップするだけ** で起動できます。

ログインすると「きょうの記録」画面が表示されます。

### A-2-1. 画面の構成

│ 2026年4月21日(火)                   │

├─────────────────────────────────────┤

│  └─ [フ] ファースト太郎 小5    未記録 │
│                                      │

### A-2-2. 色で状態を見分ける

| 表示 | 意味 |
|---|---|
| **オレンジのバッジ「未記録」** | まだ記録がない児童 |
| **緑のチェックマーク** | 記録済み |
| **「📝 紙で記入」** | 紙で書いたと誰かがマーク済み |
| **「AI未生成」**（代表者画面のみ） | 支援記録まとめがまだ入っていない |

### A-2-3. どの児童から記録する？

基本は **「未記録」に表示されている順（五十音順）** にタップして入力すればOK。
特定の児童を先に記録したい場合はタップすれば直接その児童の記録画面に行けます。

## A-3. 記録入力の流れ

### A-3-1. 児童を選ぶ

ホーム画面の「未記録」リストから該当児童をタップ。

### A-3-2. 記録入力画面が開きます

以下の項目を上から順に埋めていきます（**全部埋める必要はありません**。空欄の項目はそのままで構いません）:

1. 気分
2. 活動
3. フレーズ
4. 活動中のトピックス（自由記述）
5. 伝達事項・メモ（自由記述）
6. 支援記録まとめ（AI自動生成 → 手直し）
7. 到着時刻 / 退所時刻
8. 送迎方法

各項目の詳細は A-4 で説明します。

### A-3-3. 保存する

画面下の2つのボタンのどちらかをタップ:

- **「書き終えて次へ（代表者に転記依頼）」**（緑） — 完了扱い、次の児童へ
- **「下書き保存（この画面に留まる）」**（灰） — 保存するが未完了扱い、現画面に留まる

この2つの違いは **A-7 で詳しく** 説明します。

## A-4. 各入力項目の説明

### A-4-1. 気分（😊 / 😐 / 😢）

**3つの絵文字からタップ1回で選択**。その日のその児童の全体的な雰囲気を記録します。

- 😊 **良好**: 笑顔が多い・活発
- 😐 **普通**: 特に問題なく淡々と
- 😢 **不調**: 元気がない・泣いていた

**選ばなくてもOK**（空欄のまま保存できます）。

### A-4-2. 活動

**「活動」の下に並ぶチップを複数タップ**して、その日行った活動を選びます。

- 例: 「音読」「計算トレーニング」「ビジョントレーニング」を全部やった → 3つともタップ
- タップすると色が変わって選択状態になります
- もう一度タップすると解除
- **選んだ項目に「（詳細）」のような自由記述欄が出ることもあります**（例: 「漢字トレーニング（習っていない漢字で苦戦）」）。詳細は任意入力

### A-4-3. フレーズ

**よく使う褒め言葉や気づきを「タップで選ぶ」機能**です。

領域ごとに分かれています:
- 認知・行動
- 運動・感覚
- 言語・コミュニケーション
- 人間関係・社会性
- 生活・身辺自立

タップで複数選択可。**完全に自由記述したい場合は「伝達事項・メモ」欄**へ。

### A-4-4. 活動中のトピックス（自由記述）

**その日あった特別な出来事・エピソード** を書く欄です。

例:
- 「友達にホワイトボードの使い方を教えていた」
- 「新しい漢字で苦戦したが諦めずに取り組んだ」
- 「工作で思いがけずハサミを上手に使えた」

**音声入力が使えます**（後述 A-5）。

### A-4-5. 伝達事項・メモ（自由記述）

**保護者への連絡事項、次の担当への引継ぎ、気になったこと** を書く欄です。

例:
- 「咳が多かった。明日以降の体調要観察」
- 「明日、図工の道具を持参予定」

**音声入力が使えます**（後述 A-5）。

### A-4-6. 支援記録まとめ（AI自動生成）

**上記の入力内容から、リタリコ（h-navi）の連絡帳向けの文章をAIが自動生成**します。

- 「AI生成」ボタンをタップ → 10〜15秒で文章が出来上がる
- **必ず読んで手直し**してください（AIは事実誤認することがあります）
- 手直しした結果が最終的な記録として保存されます

詳しくは A-6 参照。

### A-4-7. 到着時刻 / 退所時刻

**児童が事業所に来た時刻・帰った時刻**を記録。

- 時刻は「時」「分」のコンボボックスで選択
- **到着時刻は入力画面を開いた時点で現在時刻が自動入力** されます（到着直後に記録を開くと便利）

### A-4-8. 送迎方法

**送迎車 / 保護者送迎 / 自力通所 / その他** から選択。

## A-5. 音声入力を使う

**スマホのマイクに向かって話すだけで文字入力できる機能**です。

### A-5-1. 使える項目

- 活動中のトピックス
- 伝達事項・メモ

### A-5-2. 使い方

1. 入力欄の横にある **🎤 マイクアイコン** をタップ
2. 最初の1回だけ「マイクの使用を許可しますか？」のダイアログが出るので **「許可」**
3. アイコンが赤くなったら話しかける（自然な話し方で OK）
4. 話し終わったら **もう一度マイクアイコンをタップ** して停止
5. 文字起こしされた文章が入力欄に反映される

### A-5-3. コツ

- **静かな場所**で話すと精度が上がります
- 話し終わってすぐ停止するより、**少し間を置いてから**停止したほうが末尾まで認識されます
- 認識ミスがあれば **手で修正** してください（完璧ではありません）
- **iPhone の Safari、Android の Chrome は問題なく動きます**

### A-5-4. 動かないとき

- マイクの使用許可を拒否してしまった場合、ブラウザ設定から再許可が必要です
- iPhone の場合: **設定 → Safari → マイク → 許可**
- 会社貸与のスマホでマイク機能が会社ポリシーで制限されていると使えないことがあります

## A-6. AI で支援記録まとめを生成する

### A-6-1. 生成手順

1. 気分・活動・フレーズ・トピックス・メモを入力した後で
2. 「支援記録まとめ」欄の上にある **「AI生成」ボタン** をタップ
3. 10〜15秒待つと、文章が自動で欄に入ります

### A-6-2. 大事なポイント：**必ず読み直す**

AI は **入力した内容から文章を組み立てるだけ** なので、以下のような誤りが起きることがあります:

- 「朝の読書は終了時刻が早かった」が「朝の読書は早く終えられた」になる（ポジティブに偏る傾向）
- 「声が大きく周囲の集中を妨げた」が「元気よく活動に参加した」になる（婉曲化の傾向）
- 記録した活動名が文中で別の表現に置き換わる

→ **AI 生成後は必ず一度読んで、事実と違う箇所は書き直してください**。

### A-6-3. AI を使わないほうが良い場面

- **重大なトラブル**（けが・激しい感情爆発・事故）— 事実関係を正確に伝える必要があるので、手書きで明確に書いてください
- **機微な情報**（家庭の事情・医療情報）— AIで文章化すると意図せず婉曲化されるリスクあり

### A-6-4. 500文字の目安

リタリコ（h-navi）の連絡帳は **500文字前後が推奨**。

- 入力画面の「支援記録まとめ」欄に**文字数カウンター**が表示されます
- 500文字を超えると **オレンジの警告** が出ます
- 超過していても保存はできますが、転記時に切れる可能性があるので注意

### A-6-5. 要約機能

文字数オーバー時に「要約」ボタンが出ます。タップすると AI が**短く書き直し**てくれます。

## A-7. 「下書き保存」と「書き終えて次へ」の違い

**ここは大事**。2つのボタンは意味が違います。

### A-7-1. 2つのボタン

記録入力画面の下に、**2つの保存ボタン** があります:

| ボタン | 色 | 動作 |
|---|---|---|
| **書き終えて次へ（代表者に転記依頼）** | 緑（目立つ） | 完了扱い、次の未記録児童に自動で移動 |
| **下書き保存（この画面に留まる）** | 灰色 | 保存するが **未完了扱い**、現画面のまま |

### A-7-2. いつどちらを使う？

**「書き終えて次へ」を使うべき場合**:

- ✅ 記録が完成した（あとで手を入れる予定なし）
- ✅ 代表者に**h-navi 連絡帳への転記を依頼してよい**状態

**「下書き保存」を使うべき場合**:

- ⏸ 中断して後で続きを書きたい
- ⏸ 他の職員の確認を受けてから完成させたい
- ⏸ AI 生成後の手直しがまだ途中

### A-7-3. 代表者側への影響

- **「書き終えて次へ」** を押した記録 → 代表者の **リタリコ転記ページに表示される** → h-navi に転記される
- **「下書き保存」** のみの記録 → 代表者の画面では「下書き」扱いで転記対象に出てこない

つまり、**「書き終えて次へ」 = 代表者に転記を依頼する合図** です。

### A-7-4. 間違えたときは

間違えて「書き終えて次へ」を押した場合、**過去記録から再度編集可能**。
ただし、代表者が既に h-navi に貼り付けてしまった場合は、h-navi 側で修正が必要になります。

## A-8. 紙で書いたときのマーク方法

紙と並行運用中に、**「この児童は今日紙で書いた」** ことをアプリに記録する方法です。

### A-8-1. 使える条件

施設長が**紙併用モード**をオンにしている事業所のみ。

オンの場合、ホーム画面の上部に **黄色の「紙併用モード 有効」案内** が表示されます。

### A-8-2. マーク手順

1. 紙で記入した児童を**普通に記録入力画面で開く**
2. 下にスクロール
3. **「紙で記入済みにする」** ボタンをタップ
4. 確認ダイアログで **「はい」**

### A-8-3. マーク後の見え方

- ホーム画面で「📝 紙で記入」と表示される
- アプリに内容は入っていない（紙のみで管理）
- 帳票や日報にはカウントされない（紙で別管理することになる）
- 代表者画面の「リタリコ転記」にも出ない（紙から手で転記）

### A-8-4. 解除したい場合

紙マーク後にアプリでも入力したくなったら:
- その児童の記録入力画面で **気分・活動などを入力して通常保存** すれば紙マーク解除
- 通常の記録として扱われるようになります

## A-10. オフライン時の挙動

**電波が届かない場所でも、ぱっと記録は使えます**。

### A-10-1. オフラインとは

- Wi-Fi も4G/5Gも繋がらない状態
- 地下駐車場、電波の弱い屋外、飛行機モード中、など

### A-10-2. オフラインになったら起こること

#### 画面の変化

- 画面最上部に **オレンジのバナー**:
  「⚠ オフライン中 — 保存は通信復帰時に自動実行されます（待機中 0 件）」

#### できること

- ✅ 記録入力（書くこと）
- ✅ 「下書き保存」「書き終えて次へ」で保存（端末内に保存される）
- ✅ 過去に**見たことのあるページ**は引き続き表示される
  - ホーム、児童一覧、記録入力画面、フレーズ、活動マスタ など
- ✅ 児童一覧・過去記録（その日開いたもの）を見ることができる

#### できないこと

- ❌ **AI 生成**（インターネット経由の処理なので）
- ❌ **音声入力**（端末によっては使えるが、基本はネット必須）
- ❌ 初めて開くページへの移動（まだ画面をキャッシュしていないため）

### A-10-3. オフライン中に保存したらどうなる？

- **端末内に一時保存**されます
- バナーに「待機中 1 件」のように待機件数が表示される
- **電波が戻った瞬間、自動で送信される**
- 送信中はバナーが青くなり「⏳ 同期中（残り 1 件）」と表示
- 全て同期できたらバナーが消える

### A-10-4. 気をつけること

- **電波の戻ったあと、アプリを閉じずに少し待つ**（自動同期が走るまで）
- 自動同期中はバナー（青）が消えるまで待つ
- 「待機中」のまま端末を壊すと、そのデータは失われる恐れがあります（電波が戻ったらまず同期完了を確認）

### A-10-5. オフラインでうっかり同じ記録を2回入力しても大丈夫？

**同じ児童の同じ日の記録は1件として管理される**ので、2重入力にはなりません。
後から書いたほうが上書き保存されます。

## B-3. リタリコ（h-navi）への転記作業

リタリコを使っている事業所向けの **主力機能** です。

### B-3-1. 基本の流れ

```
職員が入力・書き終えて次へ を押す
  ↓
代表者が /transcribe 画面を開く
  ↓
児童ごとの記録を順番に h-navi へコピペ
  ↓
h-navi 側で「登録」を押す
  ↓
代表者が「h-navi 転記完了をマーク」を押す
```

### B-3-2. 転記画面を開く

サイドバー「リタリコ転記」をクリック。

### B-3-3. 画面の見方

```
┌───────────────────────────────────────────────┐
│ 提供日: [2026-04-21▼]                         │
│ [未転記(3)] [転記済み(5)] [下書き(1)] [すべて] │
├───────────────────────────────────────────────┤
│ [未転記] 山本はる  #12345  記録者: 田中        │
│   提出: 14:30  502/500字  ⚠                   │
│   ────────────────────────                    │
│   【取組内容】                                │
│   音読／計算トレーニング                       │
│                                               │
│   【活動の様子】                              │
│   本日は音読で、声に抑揚を付けて...           │
│                                               │
│   【その他】                                  │
│   宿題も集中して取り組めた                    │
│                                               │
│   担当：田中 花子                             │
│   ────────────────────────                    │
│   [転記用コピー] [h-navi 転記完了をマーク]    │
└───────────────────────────────────────────────┘
```

### B-3-4. 日付を切り替える

画面上の **「提供日」の日付ピッカー** で別の日に切り替え可能。
過去の日付で「未転記」が残っていたら、まだ転記してない記録があるサイン。

### B-3-5. フィルタ

4つのタブで表示を絞り込めます:

- **未転記** — 職員が「書き終えて次へ」した、まだ h-navi に貼り付けていない記録（★通常作業はここ）
- **転記済み** — 既に h-navi に貼り付け済み
- **下書き** — 職員がまだ「書き終えて次へ」を押していない途中記録（転記対象外）
- **すべて** — 全件表示

### B-3-6. 児童名の横のバッジ

| 表示 | 意味 |
|---|---|
| **青の「#12345」** | その児童の h-navi userCode（児童管理番号） — 転記時の目視確認用 |
| **オレンジの「管理番号 未設定」** | 児童管理番号が未入力。同姓同名や同じ誕生日の児童がいると取り違えリスク |

**「管理番号 未設定」が出た場合** は、取り違えないよう慎重に。後で児童編集画面から userCode を入力すれば以降出なくなります。

### B-3-7. 転記手順（1件ずつ）

1. **画面上の「h-navi を別タブで開く ↗」** ボタンをクリック → 新しいタブで h-navi が開く
2. h-navi にログイン（初回のみ）→ 該当児童を検索
3. ぱっと記録側に戻って、**「転記用コピー」** ボタンをクリック（クリップボードにコピーされる）
4. h-navi の連絡帳の **「活動の様子」欄** に貼り付け（Ctrl+V）
5. h-navi 側で「登録」
6. ぱっと記録側の **「h-navi 転記完了をマーク」** をクリック → 緑色の「転記済み」状態になる

### B-3-8. 文字数超過（500字オーバー）

一番右の文字数カウンターが **502/500字 のようにオレンジ表示** されていたら超過中。

- 職員さんに「書き直し」を依頼するか
- 施設長判断で **そのまま h-navi に貼り付け**（h-navi 側で末尾が切れる可能性あり）

頻繁に超過する場合、職員さんに **AI要約機能**（A-6-5）の活用を促してください。

### B-3-9. マークを間違えて押した場合

「h-navi 転記完了をマーク」を誤って押した場合:
- 同じボタンが **「転記済み解除」** に変わっています
- クリックすれば「未転記」状態に戻ります

### B-3-10. 代表者向け運用のコツ

- **毎日の帰宅前 or 翌朝に一括処理** するリズムが最も効率的
- 職員さんが「書き終えて次へ」を押しているか、ダッシュボード画面でまず確認
- 「AI未生成」が残っている場合、転記の前に職員さんに声かけ

# Part C：共通

## C-1. トラブルシューティング

### C-1-1. ログインできない

**症状**: ログイン画面でパスワードを入れても「認証エラー」と出る

**対処**:
1. **メールアドレスのタイプミスがないか確認**（全角スペース混入に注意）
2. **パスワードの大文字・小文字を確認**
3. **仮パスワードを使っていた場合、既に変更されていないか施設長に確認**
4. それでも駄目なら開発担当者に連絡（パスワードリセットが必要）

### C-1-2. ホーム画面に児童が1人も出ない

**症状**: 「児童が登録されていません」と出る

**対処**:
1. **施設長に「児童登録が終わっているか」確認**
2. **オフライン状態になっていないか確認**（画面上部のバナーをチェック）
3. ページを一度リロード（下に引っ張って更新）

### C-1-3. 記録の「AI生成」ボタンが動かない

**症状**: ボタンを押しても何も起きない、またはエラーが出る

**対処**:
1. **オフラインでは使えません**。電波状態を確認
2. **必須項目（気分・活動・メモのいずれか）が最低1つ入力済みか確認**
3. しばらく待って再度試す（AI側が混雑している可能性）
4. 頻発するなら開発担当者に連絡

### C-1-4. 音声入力が認識されない

**症状**: マイクアイコンを押しても反応しない、または文字が出ない

**対処**:
1. **マイクの使用許可を与えているか確認**
   - iPhone: 設定 → Safari → マイク → 許可
   - Android: 設定 → アプリ → Chrome → 権限 → マイク → 許可
2. **ブラウザが古い場合は更新**（iOS/Android の OS アップデート、Chrome/Safari の最新版）
3. **イヤホン接続中** はイヤホンのマイクが使われるため、抜くと改善することも

### C-1-5. 画面が真っ白 or 読み込み中のまま

**症状**: 起動しても画面が真っ白、「読み込み中」から進まない

**対処**:
1. **ページを下に引っ張ってリロード**
2. **ブラウザのキャッシュをクリア**
   - iPhone Safari: 設定 → Safari → 履歴とウェブサイトデータを消去
   - Chrome: 設定 → プライバシー → 閲覧履歴データの削除
3. **別の WiFi or モバイル回線に切り替え**
4. それでも解決しない場合、一度ログアウト → 再ログイン

### C-1-6. オフラインから戻ったのに同期されない

**症状**: 電波が復帰してもバナーに「待機中 X 件」が残り続ける

**対処**:
1. **アプリを完全に閉じて再起動**（ホームボタンでタスクスワイプ → 再度アイコンタップ）
2. **インターネット接続を再確認**（ブラウザで別サイトが開けるか）
3. **ログアウト → 再ログイン** を試す
4. それでも戻らない場合、**データが失われる前に** 開発担当者に連絡

### C-1-7. リタリコ転記画面で児童が出てこない

**症状**: 事業所長ダッシュボードの「リタリコ転記」で何も表示されない

**対処**:
1. **日付が間違っていないか確認**
2. **フィルタが「下書き」になっていないか**（「未転記」または「すべて」に切り替え）
3. **職員がまだ「書き終えて次へ」を押していない** 可能性 — ダッシュボード画面の「AI未生成」「未記録」リストで状況確認

### C-1-8. h-navi 側に貼り付けた後、文字が崩れる

**症状**: コピーした記録を h-navi に貼り付けたら改行や【】が壊れている

**対処**:
1. **ぱっと記録側の「転記用コピー」ボタンを使っているか確認**（単純な選択&Ctrl+C ではない）
2. **h-navi 側のテキストエリアが「リッチテキスト」モードになっていないか確認**
3. それでも崩れる場合、h-navi 側の入力仕様が変わった可能性があるので開発担当者に連絡

### C-1-9. 児童管理番号を入れたのにバッジが「未設定」のまま

**症状**: 児童編集画面で児童管理番号を入力したのに、転記画面のバッジが灰色のまま

**対処**:
1. **保存ボタンを押したか確認**
2. **転記画面をリロード**
3. **別の児童と間違えていないか確認**（名前で再度確認）

### C-1-10. スマホのホーム画面から起動したら空白

**症状**: 「ホーム画面に追加」したアイコンをタップすると画面が真っ白

**対処**:
1. **一度アイコンを削除**（長押し → 削除）
2. **ブラウザで普通に開いてログイン**
3. **改めて「ホーム画面に追加」を再実行**

## C-2. よくある質問（運用編）

### Q1. 記録したけど、やっぱりもう一度 AI 生成をやり直したい

**A.** 「AI生成」ボタンを **もう一度押す** と再生成されます。ただし **前の内容は上書き** されるので、必要なら生成前にコピーしておきましょう。

### Q2. 同じ職員が2台のスマホで使える？

**A.** 使えます。同じメールアドレスとパスワードで両方からログイン可能。**直近の記録はどちらの端末からも見えます**。

### Q3. 児童管理番号を後から入れたら、過去の記録にも反映される？

**A.** 過去記録の内容自体は変わりませんが、**今後その児童の転記バッジが「未設定」から「#XXX」に変わります**。過去の転記済み記録には影響しません。

### Q4. 職員が記録を間違えて別の児童に書いた

**A.**
1. 誤入力のあった児童の記録を **削除**
2. 正しい児童を開いて **新規に入力**
3. 代表者が h-navi に既に転記済みの場合は、h-navi 側でも修正が必要

### Q5. 月の途中で紙併用モードを切り替えた

**A.** 問題ありません。切り替え前のデータはそのまま残り、以降の記録から新モードが適用されます。

### Q6. 休日や祝日も記録できる？

**A.** できます。日付に制限はありません。

### Q7. 未来の日付で記録できる？

**A.** 技術的には可能ですが、**運用上は非推奨**。出欠実績や帳票の整合性が狂います。

### Q8. 過去記録は何年前まで残る？

**A.** 削除しない限り **無期限で保存** されます。現実的には年度終わりに一括エクスポートしておくと安心。

### Q9. 月末の集計に漏れがないか確認する方法は？

**A.**
1. 事業所長ダッシュボードで月末日を選択
2. 「未記録」「AI未生成」のリストをチェック
3. リタリコ転記ページで「未転記」が残っていないか確認

### Q10. 法人として複数事業所で使える？

**A.** 現時点は **1事業所につき1環境**。複数事業所運営の場合は開発担当者にご相談ください。

### Q11. 個人情報の扱いが気になる

**A.**
- 通信はすべて HTTPS で暗号化
- データベースは **事業所ごとに完全分離**（他事業所のデータが見えない仕組み）
- 退職者アカウントは即停止可能
- AI 生成時に使う情報は、氏名・目標・活動などの最小限
- より詳しい仕様は開発担当者に **情報セキュリティ説明書** を依頼してください

### Q12. スマホを紛失したらどうすれば？

**A.**
1. 直ちに開発担当者に **アカウント停止** を依頼
2. 別端末で再ログインして業務継続
3. 紛失端末内の「オフライン待機中」データは失われる可能性あり

### Q13. 職員の異動があった

**A.**
1. 新任の方のアカウント追加 → 開発担当者に依頼
2. 異動する方のアカウント停止 → 開発担当者に依頼（当日で OK）
3. 異動する方が書いた過去記録は保持されます

## C-3. 用語集

| 用語 | 意味 |
|---|---|
| **PWA** | ウェブサイトをアプリのように使える技術。ホーム画面に追加するとネイティブアプリに近い体験 |
| **オフライン** | インターネットに繋がっていない状態 |
| **同期** | 端末に貯まった情報をサーバーに送ること（またはその逆） |
| **AI生成** | 入力内容から自然な日本語の支援記録まとめを AI が自動で書くこと |
| **活動マスタ** | 記録入力画面で選べる活動項目の一覧（事業所ごとに設定） |
| **フレーズバンク** | よく使う文章のストック。タップで選んで記録に追加できる |
| **リタリコ（h-navi）** | 放課後等デイサービス業界で使われる業務管理 SaaS。連絡帳機能を持つ |
| **userCode** | h-navi における児童の一意な識別番号。ぱっと記録では「児童管理番号」として扱う |
| **下書き保存** | 記録を途中で保存する操作。完了扱いにはならない |
| **書き終えて次へ** | 記録を完成させる操作。代表者への転記依頼の合図 |
| **紙併用モード** | 紙とアプリを並行運用する期間向けの設定 |
| **転記** | ぱっと記録の内容を h-navi 連絡帳にコピー&貼り付けする作業 |
| **業務日誌** | 1日の全児童の活動を1枚にまとめた日報 |
| **サービス提供記録** | 1児童ごとの期間内記録をまとめた帳票 |

## C-4. キーボード・タッチ操作のコツ

### タッチ操作

| 操作 | 意味 |
|---|---|
| **タップ** | 1回だけ軽く触れる（ボタン押しや選択） |
| **長押し** | 指を離さず1秒以上押し続ける（メニュー呼び出し） |
| **下にスワイプ** | 画面を下にひっぱる（リロード・更新） |
| **ピンチ** | 2本指で広げる・閉じる（拡大・縮小） |

| ショートカット | 用途 |
|---|---|
| **Tab** | 次の入力欄に移動 |
| **Shift + Tab** | 前の入力欄に戻る |
| **Ctrl + C / Cmd + C** | コピー |
| **Ctrl + V / Cmd + V** | 貼り付け |
| **F5 / Cmd + R** | ページ再読み込み |
