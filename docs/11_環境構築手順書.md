# 環境構築手順書

## 1. 概要

本書は「ぱっと記録」をゼロから構築・デプロイするための手順を網羅する。既存のコードベースが失われた場合でも、他の設計書と合わせてアプリを復元できることを目的とする。

---

## 2. 前提条件

### 2.1 必要なツール

| ツール | バージョン | 用途 |
|---|---|---|
| Node.js | 20.x 以上 | ランタイム |
| npm | 10.x 以上 | パッケージ管理（workspaces 使用） |
| Git | 最新 | バージョン管理 |
| Vercel CLI | 最新 | デプロイ |

### 2.2 必要なアカウント

| サービス | プラン | 用途 |
|---|---|---|

| Anthropic | 有料 | Claude API（AI記録生成） |

## 3. リポジトリ構成

### 3.1 モノレポ構造

```
day-service-app/
├── apps/
│   ├── main/          # メインアプリ（Next.js）
│   └── admin/         # 管理画面（Next.js）
├── packages/
│   └── shared/        # 共有ライブラリ
├── supabase/
│   └── migrations/    # DBマイグレーション
├── docs/              # 設計ドキュメント
├── package.json       # ルート（workspaces 定義）
└── .env.local         # 環境変数（gitignore対象）
```

### 3.2 npm workspaces 設定

ルート `package.json`:

```json
{
  "name": "day-service-app",
  "private": true,
  "workspaces": ["apps/*", "packages/*"],
  "scripts": {
    "dev:main": "npm run dev --workspace=@patto/main",
    "dev:admin": "npm run dev --workspace=@patto/admin",
    "build:main": "npm run build --workspace=@patto/main",
    "build:admin": "npm run build --workspace=@patto/admin",
    "lint:main": "npm run lint --workspace=@patto/main",
    "lint:admin": "npm run lint --workspace=@patto/admin"
  }
}
```

### 3.3 パッケージ名

| パッケージ | npm name | ポート |
|---|---|---|
| メインアプリ | `@patto/main` | 3000 |
| 管理画面 | `@patto/admin` | 3001 |
| 共有ライブラリ | `@patto/shared` | — |

## 4. Supabase セットアップ

### 4.1 プロジェクト作成

1. [Supabase Dashboard](https://supabase.com/dashboard) にログイン
2. "New Project" をクリック
3. 設定:
   - **Name:** patto-kiroku（任意）
   - **Database Password:** 強力なパスワードを設定
   - **Region:** Northeast Asia（ap-northeast-1）
   - **Pricing Plan:** Free（開始時）

### 4.2 マイグレーション実行

Supabase Dashboard の SQL Editor で以下の順序で実行:

| 順序 | ファイル | 内容 |
|---|---|---|
| 1 | `001_initial_schema.sql` | テーブル定義、RLS ポリシー、トリガー |
| 2 | `002_seed_phrases.sql` | デフォルトフレーズ（20件）の INSERT |
| 3 | `003_settings_phase2.sql` | 施設更新・スタッフ管理用 RLS 追加 |
| 4 | `004_signup_policies.sql` | サインアップ用 INSERT ポリシー |
| 5 | `005_daily_records_unique.sql` | daily_records のユニーク制約追加 |
| 6 | `006_admin_features.sql` | super_admin、施設拡張カラム、管理用 RLS |
| 7 | `007_ai_text_column.sql` | daily_records に ai_text カラム追加 |

**重要:** マイグレーションは必ず番号順に実行すること。

### 4.3 認証設定

Supabase Dashboard → Authentication → Settings:

1. **Email Auth:** 有効化（デフォルト）
2. **Email Confirmations:** 無効化（開発時。本番では要検討）
3. **Minimum Password Length:** 6文字

### 4.4 API キー取得

Supabase Dashboard → Settings → API:

| キー | 用途 | 環境変数 |
|---|---|---|
| Project URL | Supabase エンドポイント | `NEXT_PUBLIC_SUPABASE_URL` |
| anon (public) key | クライアント用 | `NEXT_PUBLIC_SUPABASE_ANON_KEY` |
| service_role key | サーバー用（RLS バイパス） | `SUPABASE_SERVICE_ROLE_KEY` |

## 6. ローカル開発環境

### 6.1 リポジトリクローン

```bash
git clone https://github.com/{owner}/day-service-app.git
cd day-service-app
```

### 6.2 依存関係インストール

```bash
npm install
```

ルートから実行すると、workspaces 内の全パッケージの依存関係がインストールされる。

### 6.3 環境変数設定

ルートに `.env.local` を作成:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here
ANTHROPIC_API_KEY=sk-ant-your-key-here
ADMIN_PASSWORD=your-admin-password-here
```

**注意:** `.env.local` は各アプリのディレクトリではなく、ルートに配置する。Next.js は親ディレクトリの `.env.local` も読み込む。

### 6.4 開発サーバー起動

```bash
# メインアプリ（port 3000）
npm run dev:main

# 管理画面（port 3001）— 別ターミナル
npm run dev:admin
```

### 6.5 動作確認

| URL | 画面 |
|---|---|
| http://localhost:3000 | メインアプリ（ログイン画面） |
| http://localhost:3000/login | メインアプリ ログイン |
| http://localhost:3001 | 管理画面（ログイン画面） |

## 8. Vercel デプロイ

### 8.1 プロジェクト構成

2つの Vercel プロジェクトを作成:

| プロジェクト | ルートディレクトリ | ドメイン |
|---|---|---|
| day-service-app | `apps/main` | day-service-app.vercel.app |
| day-service-admin | `apps/admin` | day-service-admin.vercel.app |

### 8.2 Vercel プロジェクトリンク

```bash
# ルートディレクトリで実行
npx vercel link
```

`.vercel/project.json` が生成される。

### 8.3 環境変数設定（Vercel Dashboard）

各プロジェクトの Settings → Environment Variables に以下を設定:

**メインアプリ:**
| 変数 | スコープ |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Production, Preview, Development |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Production, Preview, Development |
| `SUPABASE_SERVICE_ROLE_KEY` | Production, Preview |
| `ANTHROPIC_API_KEY` | Production, Preview |

**管理画面:**
| 変数 | スコープ |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Production, Preview, Development |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Production, Preview, Development |
| `SUPABASE_SERVICE_ROLE_KEY` | Production, Preview |
| `ADMIN_PASSWORD` | Production, Preview |

### 8.4 vercel.json 設定

各アプリの `vercel.json`:

```json
{
  "installCommand": "cd ../.. && npm install",
  "framework": "nextjs"
}
```

**重要:** モノレポのため、`installCommand` でルートに移動してからインストールする必要がある。

### 8.5 デプロイ実行

```bash
# ルートディレクトリから（メインアプリ）
npx vercel --prod --yes
```

**注意:**
- `apps/main` 単体からの `vercel` コマンドは失敗する
- 必ずルートディレクトリから実行すること

### 8.6 GitHub 連携（自動デプロイ）

1. Vercel Dashboard → Project → Settings → Git
2. GitHub リポジトリを接続
3. Root Directory に `apps/main` または `apps/admin` を設定
4. main ブランチへの push で自動デプロイ

## 9. Next.js 設定

### 9.1 next.config.ts（共通）

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  transpilePackages: ["@patto/shared"],
};

export default nextConfig;
```

`@patto/shared` はビルド済みパッケージではないため、`transpilePackages` で指定が必要。

### 9.2 TypeScript 設定（共通）

```json
{
  "compilerOptions": {
    "target": "ES2017",
    "strict": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

### 9.3 Tailwind CSS 4 設定

`postcss.config.mjs`:
```javascript
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
export default config;
```

カスタムカラーは `globals.css` の `@theme` ブロックで定義（Tailwind CSS 4 の新構文）。

## 11. トラブルシューティング

### 11.1 よくある問題

| 問題 | 原因 | 解決策 |
|---|---|---|
| `Module not found: @patto/shared` | transpilePackages 未設定 | next.config.ts に追加 |
| Vercel デプロイ失敗 | apps/main から直接デプロイ | ルートから `npx vercel` 実行 |
| RLS でデータ取得不可 | profiles 未作成 | サインアップフローでプロフィール作成を確認 |
| AI 生成エラー | ANTHROPIC_API_KEY 未設定 | 環境変数を確認 |
| 管理画面ログイン不可 | ADMIN_PASSWORD 未設定 | Vercel 環境変数を確認 |
| `npm install` 失敗 | workspaces 解決エラー | `node_modules` 削除後に再実行 |

### 11.2 環境変数の確認

```bash
# ローカルで環境変数が読み込まれているか確認
node -e "console.log(process.env.NEXT_PUBLIC_SUPABASE_URL)"
```

### 11.3 Supabase 接続テスト

Supabase Dashboard → SQL Editor で実行:

```sql
SELECT count(*) FROM facilities;
SELECT count(*) FROM phrase_bank WHERE is_default = true;
```

デフォルトフレーズが 20件 あれば seed 適用済み。

## 12. 初期データ投入

### 12.1 最初の管理者アカウント作成

1. メインアプリ（localhost:3000）にアクセス
2. 「新規登録」タブを選択
3. メール、パスワード、表示名、施設名を入力して登録
4. 自動的に admin ロールのユーザーが作成される

### 12.2 スーパー管理者の設定

Supabase Dashboard → SQL Editor:

```sql
UPDATE profiles SET is_super_admin = true WHERE display_name = '管理者名';
```

### 12.3 管理画面へのアクセス

1. 管理画面（localhost:3001）にアクセス
2. `ADMIN_PASSWORD` に設定したパスワードでログイン
