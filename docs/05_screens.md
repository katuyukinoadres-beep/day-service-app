# 画面・UI設計書

> プロジェクト: **ぱっと記録（Patto Kiroku）**  |  docs-template v1.0 準拠

---

## 1. デザイン原則

- **シンプルさ優先**: 1画面1タスク。記録入力は最小タップ数で完了できるよう、チップ選択・ワンタップ操作を基本とする
- **モバイルファースト**: スマートフォン（375px〜）での操作を最優先に設計。BottomNav による片手操作を重視する
- **即時フィードバック**: 保存成功時のトースト表示、AI生成中のスピナー表示など、操作に対して即座に視覚的反応を返す
- **一貫性**: 全画面で同じカラーパレット・コンポーネント・操作パターンを使用。ボタンの位置と配色を統一する
- **業務効率**: 保存後の自動遷移（次の未記録児童へ）により、連続記録のフローを最適化する

### 2.1 カラーパレット

アプリで使用するカラーを以下に定義する。

| 用途 | CSS変数 / クラス | 色コード | 使用場面 |
|------|-----------------|---------|---------|
| プライマリ | `--primary` | `#1B6B4A` | CTAボタン、アクティブタブ、プログレスバー |
| プライマリ（淡） | `--primary-light` | `#E8F5EE` | 選択状態の背景、チップ選択時 |
| アクセント | `--accent` | `#E8913A` | 未記録バッジ、警告表示 |

### 2.2 タイポグラフィ

テキストスタイルの定義。フォントは Noto Sans JP を使用する。

| 要素 | サイズ | ウェイト | 用途 |
|------|--------|---------|------|
| ヘッダータイトル | 18px | Bold (700) | 画面タイトル |
| セクションタイトル | 16px | Bold (700) | セクション見出し |
| 本文 | 15px | Normal (400) | メインテキスト |
| ラベル | 14px | Medium (500) | フォームラベル、カードサブ情報 |
| サブテキスト | 13px | Normal (400) / Medium (500) | 補助テキスト、タイムスタンプ |
| バッジテキスト | 12px | Medium (500) | バッジ、チップ内テキスト |

### 2.3 スペーシング・形状

**スペーシング:**

| トークン | 値 | 用途 |
|---------|---|------|
| `space-xs` | 4px | アイコンとラベルの間 |
| `space-sm` | 8px | リスト項目間、チップ間 |
| `space-md` | 16px | カード内パディング、ページ左右パディング |
| `space-lg` | 24px | セクション間の余白 |
| `space-xl` | 32px | 画面上下のパディング |

**形状:**

| 要素 | 角丸 | 影 |
|------|------|---|
| ボタン | 12px (`rounded-xl`) | なし |
| カード | 16px (`rounded-2xl`) | `shadow-sm` |
| モーダル | 16px (`rounded-2xl`) | `shadow-lg` |
| 入力フィールド | 12px (`rounded-xl`) | なし |
| チップ | 9999px (`rounded-full`) | なし |
| アバター | 9999px (`rounded-full`) | なし |

**タップターゲット:** 最小 44px x 44px（Apple HIG 準拠）

### 2.4 レスポンシブ設計

モバイルファーストで設計する。デフォルトスタイルはモバイル向け。

| ブレイクポイント | 画面幅 | 想定デバイス | 主な変更点 |
|---------------|--------|------------|-----------|
| デフォルト | 〜639px | スマートフォン | 1カラム、BottomNav表示 |
| `sm` | 640px〜 | 大型スマホ | 基本レイアウト維持 |
| `md` | 768px〜 | タブレット | 帳票のテーブル表示最適化 |
| `lg` | 1024px〜 | デスクトップ | max-width で中央配置 |

管理画面（`apps/admin`）は `md` 以上でサイドバーナビ（`w-64` 固定）、モバイルではハンバーガーメニュー + オーバーレイに切り替える。

### 5.1 SCR-001: ログイン / 新規登録

**目的**: メールアドレスとパスワードで認証し、ホーム画面に遷移する。新規登録時は施設とプロフィールを同時に作成する。

**レイアウト:**

```
+----------------------------------+
|           (余白 48px)            |
|          [ぱ] ぱっと記録          |
|      放課後等デイサービス支援記録   |
|           (余白 32px)            |
|  [ログイン | 新規登録]  タブ       |
|  +----------------------------+  |
|  | メールアドレス               |  |
|  +----------------------------+  |
|  +----------------------------+  |
|  | パスワード                   |  |
|  +----------------------------+  |
|  (signup時: 施設名、表示名)      |
|           (余白 8px)             |
|  [        ログイン         ]     |
+----------------------------------+
```

**表示要素:**

| 要素 | 内容 | データソース |
|------|------|------------|
| ロゴ | 「ぱ」アイコン + アプリ名 | 静的アセット |
| タブ切替 | 「ログイン」/「新規登録」、アクティブは `border-b-2 border-primary` | ユーザー操作 |
| メールアドレス入力 | `type="email"`, `required` | ユーザー入力 |
| パスワード入力 | `type="password"`, 6文字以上, `required` | ユーザー入力 |
| 施設名入力（新規登録時） | `type="text"`, `required` | ユーザー入力 |
| 表示名入力（新規登録時） | `type="text"`, `required` | ユーザー入力 |
| 送信ボタン | プライマリボタン。`loading` 中は「ログイン中...」 | - |

**ユーザー操作:**

| 操作 | 動作 |
|------|------|
| タブを切り替え | ログイン / 新規登録モードを切り替え |
| 送信ボタンをタップ | 認証API呼び出し。成功 → `/` に遷移。失敗 → エラー表示 |
| 新規登録を送信 | サインアップ → 施設作成 → プロフィール作成 → `/` に遷移 |

**状態:**

| 状態 | 表示 |
|------|------|
| ローディング中 | ボタンテキストが「ログイン中...」に変化、ボタン `disabled` |
| データなし | 該当なし（初期表示が空フォーム） |
| エラー | フォーム上部に赤背景（`bg-red-50 text-red-600`）でエラーメッセージ |
| オフライン | 該当なし（オフライン対応なし） |

### 5.4 SCR-004: 日次記録入力

**目的**: 児童ごとの支援記録（気分・活動・フレーズ・メモ・AI文章・送迎方法）を入力し保存する。

**レイアウト:**

```
+----------------------------------+
| [<-] [G] 田中太郎     [編集中]    |
+----------------------------------+
| 来所時刻 [14:30]  退所時刻 [    ] |
|                                  |
| きょうの気分                     |
| [😊良好] [😐普通] [😢不調]       |
|                                  |
| 活動内容                         |
| [工作][運動][学習][自由遊び]      |
| [SST][おやつ][外出][音楽][調理]   |
|                                  |
| 記録フレーズ                     |
|  ▼ 認知・行動                    |
|  [v フレーズテキスト...]          |
|  [  フレーズテキスト...]          |
|                                  |
| メモ（自由記入）                 |
| [テキストエリア rows=3]           |
|                                  |
| 本日の支援記録まとめ             |
| [AIで下書き作成 v2.0]            |
| [テキストエリア rows=6]           |
|                                  |
| 送迎方法                         |
| [送迎車][保護者][自力][その他]     |
|                                  |
| [     保存して次へ          ]     |
+----------------------------------+
```

**表示要素:**

| 要素 | 内容 | データソース |
|------|------|------------|
| ヘッダー | 戻るボタン + 児童アイコン + 名前 | `children` テーブル |
| 編集中バッジ | 既存記録がある場合に表示 | `daily_records` テーブル |
| 来所/退所時刻 | `type="time"` 入力。新規時は来所に現在時刻をプリセット | ユーザー入力 |
| 気分 | 3択ボタン（`good` / `neutral` / `bad`）。選択中は `border-primary bg-primary-light` | ユーザー入力 |
| 活動チップ | 9種類のトグル式 Chip。複数選択可 | 定数 `ACTIVITIES` |
| フレーズ | カテゴリ別グループ表示。児童の `domain_tags` に一致するフレーズを優先・黄色ハイライト | `phrase_bank` テーブル |
| メモ | `textarea`（`rows=3`） | ユーザー入力 |
| AI生成ボタン | 「AIで下書き作成 v2.0」、`generating` 中はスピナー | `/api/generate-record` |
| 支援記録まとめ | `textarea`（`rows=6`）。AI 生成後に表示、手動編集可 | AI 生成 or ユーザー入力 |
| 送迎方法 | 4択の排他選択 Chip | ユーザー入力 |
| 保存ボタン | 新規「保存して次へ」/ 編集「更新する」。`saving` 中は「保存中...」 | - |

**ユーザー操作:**

| 操作 | 動作 |
|------|------|
| 気分ボタンをタップ | 選択状態をトグル。再タップで解除（`null` に戻る） |
| 活動チップをタップ | 複数選択のトグル |
| フレーズをタップ | チェックマーク付きハイライト。`phrases` 配列に追加/削除 |
| 「AIで下書き作成」をタップ | 入力内容 + 児童情報を Claude API に送信。生成テキストを支援記録まとめ欄に挿入 |
| 「保存して次へ」をタップ | 記録を保存。次の未記録児童がいれば自動遷移、なければ `/` に遷移 |

**状態:**

| 状態 | 表示 |
|------|------|
| ローディング中 | 画面全体のローディング表示 |
| データなし | 該当なし（フォーム初期状態） |
| エラー | 保存失敗時にアラート表示 |
| オフライン | 該当なし |

### 6.1 `<Button>` --- 汎用ボタン

**Props:**

| Prop | 型 | 必須 | デフォルト | 説明 |
|------|----|------|----------|------|
| `variant` | `'primary' \| 'secondary' \| 'danger' \| 'ghost'` | No | `'primary'` | ボタンの見た目 |
| `fullWidth` | `boolean` | No | `false` | 横幅 100% |
| `disabled` | `boolean` | No | `false` | 非活性状態 |
| `children` | `ReactNode` | Yes | - | ボタンテキスト |

**動作:**

- `disabled={true}` の場合: `opacity-50 cursor-not-allowed` を適用

**表示条件:**

| variant | 背景色 | テキスト色 | ボーダー |
|---------|--------|----------|---------|
| primary | `bg-primary` (`#1B6B4A`) | `text-white` | なし |
| secondary | `bg-gray-100` | `text-gray-700` | `border-gray-300` |
| danger | `bg-danger` (`#DC2626`) | `text-white` | なし |
| ghost | 透明 | `text-primary` | なし |

### 6.2 `<Card>` --- 汎用カード

**Props:**

| Prop | 型 | 必須 | デフォルト | 説明 |
|------|----|------|----------|------|
| `padding` | `boolean` | No | `true` | 内部パディング（16px）の有無 |
| `className` | `string` | No | `''` | 追加CSSクラス |

**表示:** `rounded-2xl`（16px）+ `shadow-sm` + `bg-white`

### 6.3 `<Chip>` --- 選択チップ

**Props:**

| Prop | 型 | 必須 | デフォルト | 説明 |
|------|----|------|----------|------|
| `label` | `string` | Yes | - | 表示テキスト |
| `selected` | `boolean` | No | `false` | 選択状態 |
| `onToggle` | `() => void` | Yes | - | トグルハンドラ |
| `size` | `'sm' \| 'md'` | No | `'md'` | サイズ |

**動作:**

- 選択時: `bg-primary-light border-primary text-primary` を適用
- 非選択時: `bg-gray-100 border-transparent text-gray-700` を適用
- `rounded-full`（9999px）

### 6.4 `<Input>` / `<Textarea>` --- 入力フィールド

**Props:**

| Prop | 型 | 必須 | デフォルト | 説明 |
|------|----|------|----------|------|

| `error` | `string` | No | - | エラーメッセージ |
| (HTML attributes) | - | - | - | 標準 input/textarea 属性 |

**動作:**

- エラー時: ボーダーを `border-danger` に変更、エラーメッセージを `text-danger` で表示

**Props:**

### 6.6 `<BottomNav>` --- ボトムナビゲーション
