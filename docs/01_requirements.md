> プロジェクト: **ぱっと記録（Patto Kiroku）**  |  docs-template v1.0 準拠

| 項目 | 値 |
|------|---|
| プロジェクト名 | ぱっと記録（Patto Kiroku） |
| 概要 | 放課後等デイサービスの支援記録をデジタル化し、スタッフの記録業務を効率化するWebアプリ |

## 1. 背景と目的

### 1.1 背景

放課後等デイサービス事業所では、日々の児童の支援記録を紙ベースで管理している施設が多く、記録作業に多大な時間を要している。記録の漏れや転記ミス、帳票作成の手間が現場のスタッフの負担となっている。

既存の業務支援ツールは汎用的なものが多く、放課後等デイサービス特有の業務フロー（支援領域タグ、フレーズバンク、児童ごとの個別支援目標管理など）に最適化されていない。また、現場スタッフはPCに不慣れなケースも多く、スマートフォンで直感的に操作できるツールが求められている。

### 1.2 目的

本システムは、放課後等デイサービスにおける日々の支援記録をデジタル化し、スタッフの記録業務を効率化するWebアプリケーションである。スマートフォンからの操作を前提としたモバイルファーストの設計により、現場での即時記録を可能にする。

Claude Sonnet 4.5 APIを活用したAI支援記録まとめ生成機能により、活動内容やフレーズから自然な支援記録文章を自動生成し、記録品質の均一化と作業時間の短縮を実現する。

### 1.3 対象ユーザー

対象ユーザーの種別と利用場面を示す。

| ユーザー種別 | 説明 | 主な利用場面 |
|-------------|------|------------|
| 管理者（admin） | 施設責任者。施設設定・スタッフ管理・全機能の利用が可能 | 施設の日次管理、スタッフ招待、帳票出力。PCまたはスマートフォンで利用 |
| スタッフ（staff） | 支援員。日常の記録入力・閲覧・帳票出力が可能 | 児童の支援記録入力（来所時・活動中・退所時）。スマートフォンで現場から即時入力 |
| スーパー管理者 | システム全体の管理者。Admin管理画面から全施設の管理が可能 | 全施設の横断管理、統計確認。PCで週次〜月次で利用 |

## 2. 機能要件

### 2.1 認証・ユーザー管理

ユーザーの登録、ログイン、施設へのスタッフ招待に関する機能。

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| AUTH-001 | 新規登録（施設作成） | メールアドレス・パスワードで新規登録。施設も同時に作成し、管理者ロールが付与される | 必須 |
| AUTH-002 | ログイン | メールアドレス・パスワードでログイン。成功後ホーム画面に遷移 | 必須 |
| AUTH-003 | ログアウト | セッションを終了しログイン画面へ遷移 | 必須 |
| AUTH-004 | スタッフ招待 | 招待リンクを生成し、新規スタッフを施設に紐付けて登録。対象ロール: admin | 必須 |
| AUTH-005 | ロール変更 | スタッフのロール（`admin`/`staff`）を切り替え。対象ロール: admin | 推奨 |

### 2.2 児童管理

施設に所属する児童の登録・編集・一覧管理に関する機能。

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| CHILD-001 | 児童一覧表示 | 施設に登録された有効な児童を五十音順で一覧表示 | 必須 |
| CHILD-002 | 児童新規登録 | 名前、ふりがな、生年月日、学校、学年、アイコン色、支援目標、支援領域タグを登録 | 必須 |
| CHILD-003 | 児童情報編集 | 既存の児童情報を編集。有効/無効の切り替えも可能（論理削除） | 必須 |

### 2.3 日次記録

児童ごとの日々の支援記録の入力・編集・閲覧に関する機能。

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| REC-001 | 本日の記録ダッシュボード | 当日の記録進捗をプログレスバーで表示。未記録/記録済みの児童を分けて表示 | 必須 |
| REC-002 | 日次記録入力 | 児童ごとに来所・退所時刻、気分、活動内容、記録フレーズ、メモ、支援記録まとめ、送迎方法を記録 | 必須 |
| REC-003 | 日次記録編集 | 既存の記録を編集（同日の再アクセスで自動復元） | 必須 |
| REC-004 | 自動遷移 | 記録保存後、次の未記録児童の記録画面へ自動遷移 | 推奨 |
| REC-005 | 記録履歴閲覧 | 日付・児童で絞り込んで過去の記録を閲覧 | 必須 |
| REC-006 | 記録削除 | 過去の記録を削除 | 推奨 |
| REC-007 | AI支援記録まとめ生成 | 入力情報（気分、活動、フレーズ、メモ、児童基本情報）をもとにClaude Sonnet 4.5 APIで支援記録文章を自動生成。直接手入力も可能。ボタンにバージョンラベル（v2.0）を表示 | 必須 |

### 2.4 フレーズバンク

記録入力を効率化するための定型文の管理機能。

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| PHR-001 | フレーズ一覧表示 | デフォルト・カスタムフレーズをカテゴリ別に表示 | 必須 |
| PHR-002 | カスタムフレーズ追加 | カテゴリ、テキスト、支援領域タグ、表示順を指定してフレーズを追加 | 推奨 |
| PHR-003 | カスタムフレーズ編集 | 既存のカスタムフレーズを編集（デフォルトフレーズは編集不可） | 推奨 |
| PHR-004 | フレーズ優先表示 | 記録画面で児童の支援領域タグに合致するフレーズをハイライト表示 | 推奨 |

### 2.5 帳票出力

支援記録の印刷・PDF出力に関する機能。

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| DOC-001 | 業務日誌 | 指定日の全児童の記録を一覧表形式で印刷/PDF出力。表示項目：児童名、来所・退所時刻、活動内容、支援記録まとめ、送迎、記録者。署名欄付き | 必須 |
| DOC-002 | サービス提供記録 | 指定日・指定児童の個別記録を帳票形式で印刷/PDF出力。保護者確認欄・管理者署名欄付き | 必須 |
| DOC-003 | 月間出席サマリー | 指定月の全児童の出席状況をカレンダー形式で表示・印刷 | 推奨 |

### 2.6 Admin管理画面（スーパー管理者向け）

システム全体のスーパー管理者向け管理機能。別アプリ（`apps/admin`）として構成。

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| ADM-001 | Admin ログイン | パスワード認証による管理画面へのアクセス | 必須 |
| ADM-002 | ダッシュボード | 全施設の統計情報（施設数、ユーザー数、記録数）を表示 | 必須 |
| ADM-003 | 施設一覧・詳細 | 全施設の一覧表示、施設詳細（スタッフ一覧、記録一覧）の閲覧 | 必須 |
| ADM-004 | 施設編集 | 施設名・プラン・メモ・有効/無効状態の変更 | 推奨 |
| ADM-005 | ユーザー一覧 | 全ユーザーの一覧表示 | 推奨 |

### 2.7 設定

ユーザー・施設の設定管理機能。

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| SET-001 | プロフィール編集 | 表示名の変更 | 推奨 |
| SET-002 | 施設設定 | 施設名の変更。対象ロール: admin | 推奨 |
| SET-003 | スタッフ管理 | 施設所属スタッフの一覧表示・ロール変更。対象ロール: admin | 推奨 |

## 3. 非機能要件

### 3.1 パフォーマンス

- 画面遷移: 1秒以内（ネットワーク遅延除く）
- データ読み込み: 2秒以内
- 同時接続ユーザー数: 施設あたり最大20名

### 3.2 可用性

- 稼働率: 99.5%以上（Vercel + Supabase SLAに準拠）
- 計画メンテナンス時のダウンタイム通知

### 3.3 セキュリティ（概要）

- 通信は全てHTTPS
- ユーザー認証（メール/パスワード）
- 施設単位のデータ分離（マルチテナント）
- Row Level Security（RLS）によるデータアクセス制御
- セッション管理（Cookieベース、`@supabase/ssr`）

### 3.4 対応環境

| 環境 | 対応範囲 |
|------|---------|
| ブラウザ | Chrome, Safari, Edge（最新2バージョン） |
| デバイス | スマートフォン（主）、タブレット、デスクトップ |
| OS | iOS 15+, Android 12+, Windows 10+, macOS 12+ |
| 画面幅 | モバイルファースト設計（デスクトップブレイクポイントなし） |

### 3.5 アクセシビリティ

- タップターゲット最小44px
- 適切なラベル付け（`aria-label`）
- 十分なコントラスト比
- セマンティックHTML使用

### 3.6 拡張性

- 新しい活動内容の追加が可能
- フレーズバンクのカスタマイズ
- 帳票テンプレートの追加

---

## 4. スコープ外

- オフライン対応（インターネット接続必須）
- ネイティブアプリ（Webアプリとして提供。PWA対応済み）
- 保護者向けポータル（スタッフ・管理者向けに特化）
- 決済・課金機能（現時点では無料提供）
- 多言語対応（日本語のみ。ユーザー基盤が日本限定のため）
- リアルタイム共同編集（同時編集の競合解決は対象外）

## 5. ドメイン定義

### 5.1 支援領域タグ

児童の支援計画やフレーズの分類に使用する5つの領域区分。

| 値 | 表示名 | 説明 |
|----|--------|------|
| `健康・生活` | 健康・生活 | 基本的な生活習慣、健康管理に関する支援領域 |
| `運動・感覚` | 運動・感覚 | 身体運動、感覚統合に関する支援領域 |
| `認知・行動` | 認知・行動 | 認知発達、行動調整に関する支援領域 |
| `言語・コミュニケーション` | 言語・コミュニケーション | 言語発達、コミュニケーション能力に関する支援領域 |
| `人間関係・社会性` | 人間関係・社会性 | 対人関係、社会的スキルに関する支援領域 |

### 5.2 活動種別

日次記録で選択可能な活動内容のカテゴリ。

| 値 | 説明 |
|----|------|
| `工作` | 工作・制作活動 |
| `運動` | 体操・スポーツ等の身体活動 |
| `学習` | 宿題・学習支援 |
| `自由遊び` | 自由時間の遊び |
| `SST` | ソーシャルスキルトレーニング |
| `おやつ` | おやつの時間 |
| `外出` | 外出活動 |
| `音楽` | 音楽活動 |
| `調理` | 調理活動 |

### 5.3 気分

日次記録で記録する児童の気分状態。

| 値 | 表示名 | 説明 |
|----|--------|------|
| `good` | 良好 | 機嫌がよく、活動的な状態 |
| `neutral` | 普通 | 特に変わりのない通常の状態 |
| `bad` | 不調 | 体調不良や気分が落ち込んでいる状態 |

### 5.4 送迎方法

児童の通所・帰宅の送迎手段。

| 値 | 説明 |
|----|------|
| `送迎車` | 事業所の送迎車を利用 |
| `保護者送迎` | 保護者による送り迎え |
| `自力通所` | 児童が自力で通所 |
| `その他` | 上記以外の方法 |

### 5.5 ユーザーロール

システム内のユーザー権限区分。

| 値 | 表示名 | 説明 |
|----|--------|------|
| `admin` | 管理者 | 施設責任者。施設設定・スタッフ管理・全機能の利用が可能 |
| `staff` | スタッフ | 支援員。日常の記録入力・閲覧・帳票出力が可能 |

## 6. 制約事項

- インターネット接続が必須（オフライン対応なし）
- 児童の物理削除は行わず、論理削除（`is_active` フラグ）で対応
- デフォルトフレーズは編集・削除不可
- 1施設あたりの児童数は実運用上100名程度を想定
- Supabase Free プラン: DB 500MB、MAU 50,000
- Vercel: 2プロジェクト構成（メインアプリ + Admin管理画面）
- Anthropic API: 従量課金（AI記録生成のコスト管理が必要）

## 7. 用語集

プロジェクト固有の用語を定義する。

| 用語 | 説明 |
|------|------|
| 放課後等デイサービス | 障害のある子どもに対し、放課後や休日に生活能力向上のための支援を行うサービス |
| 支援記録 | 児童の日々の活動、様子、特記事項を記録した書類 |
| 業務日誌 | 事業所全体の一日の活動記録をまとめた帳票 |
| サービス提供記録 | 個別の児童に対するサービス内容を記録した帳票 |
| フレーズバンク | 記録入力を効率化するための定型文の集合。デフォルト（全施設共通）とカスタム（施設固有）がある |
| 支援記録まとめ | 本日の支援記録をまとめた文章。手入力またはAI（Claude Sonnet 4.5）で下書き生成が可能 |
| Admin管理画面 | スーパー管理者用の管理ダッシュボード。全施設の横断管理に使用。`apps/admin` として独立デプロイ |
| RLS | Row Level Security。PostgreSQLのデータベースレベルで行単位のアクセス制御を行う仕組み |
| マルチテナント | `facility_id` による論理分離 + RLS で、施設ごとのデータを完全に分離する方式 |
