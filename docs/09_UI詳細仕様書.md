# UI 詳細仕様書

## 1. 概要

本書は「ぱっと記録」メインアプリおよび管理画面の全画面について、コンポーネント構成・状態管理・フォームバリデーション・ユーザーインタラクション・画面遷移を詳細に定義する。

---

## 2. メインアプリ（apps/main）

### 2.1 ルートレイアウト

| 項目 | 値 |
|---|---|
| フォント | Noto Sans JP（400, 500, 700） |
| テーマカラー | `#1B6B4A` |
| ビューポート | `width=device-width, initialScale=1, maximumScale=1` |
| PWA | `manifest.json` 参照、`apple-mobile-web-app-capable` |

### 2.2 (main) レイアウト — BottomNav 付き

- BottomNav を固定表示（z-20）
- `pb-20` でコンテンツがナビに隠れないようにする
- safe-area-inset-bottom 対応

**BottomNav 項目:**

| ラベル | パス | アイコン |
|---|---|---|
| きょう | `/` | カレンダー |
| 児童 | `/children` | ユーザーグループ |
| 帳票 | `/documents` | ドキュメント |
| 履歴 | `/history` | 時計 |
| 設定 | `/settings` | 歯車 |

- アクティブ判定: `pathname` の前方一致（`/` は完全一致）
- アクティブ色: `text-primary`、非アクティブ色: `text-gray-400`

### 2.3 ログイン画面（`/login`）

**状態管理:**
| state | 型 | 初期値 | 用途 |
|---|---|---|---|
| mode | `"login" \| "signup"` | `"login"` | タブ切り替え |
| email | string | `""` | メールアドレス |
| password | string | `""` | パスワード |
| displayName | string | `""` | 表示名（signup時） |
| facilityName | string | `""` | 施設名（signup時） |
| error | string | `""` | エラーメッセージ |
| loading | boolean | `false` | 送信中フラグ |

**フォームバリデーション:**
| フィールド | ルール |
|---|---|
| email | HTML type="email" による標準バリデーション、required |
| password | 6文字以上（クライアント側で検証）、required |
| displayName | required（signup時のみ表示） |
| facilityName | required（signup時のみ表示） |

**UI構成:**
- ロゴ（ぱ アイコン + "ぱっと記録"）
- タブ切り替え（"ログイン" / "新規登録"）— アクティブタブは `border-b-2 border-primary`
- フォームフィールド
- エラー表示: 赤背景（`bg-red-50 text-red-600`）のアラートボックス
- 送信ボタン: `loading ? "ログイン中..." : "ログイン"`

**画面遷移:**
- ログイン成功 → `/`（router.push）
- サインアップ成功 → `/`（router.push）
- 認証済みユーザーがアクセス → `/` へリダイレクト（middleware）

### 2.5 ホーム画面（`/`）— きょうの記録

**状態管理:**
| state | 型 | 初期値 |
|---|---|---|
| children | Child[] | `[]` |
| records | DailyRecord[] | `[]` |
| loading | boolean | `true` |

**データ取得:**
- children: `is_active = true`、`name_kana ASC` ソート
- daily_records: `date = 今日の日付`

**UI構成:**
1. **ヘッダー:** "きょうの記録" + 日付表示（YYYY年M月D日(曜日)）
2. **プログレス:** "X/Y 完了" テキスト + プログレスバー（`bg-primary` / `bg-gray-200`）
3. **未記録セクション:** "未記録" ラベル
   - 児童カード: アイコン（icon_color 背景 + 名前頭文字）、名前、学校/学年 → タップで `/record/[childId]`
4. **記録済みセクション:** "記録済み" ラベル + チェックマークバッジ
   - 児童カード: 気分絵文字 + 活動プレビュー

**空状態:** "児童が登録されていません" + "児童を追加する" リンク（→ `/children/new`）

**ローディング状態:** "読み込み中..."

### 2.6 記録入力画面（`/record/[childId]`）

**注意:** BottomNav 非表示（(main) レイアウト外）

**状態管理:**
| state | 型 | 初期値 |
|---|---|---|
| child | Child \| null | `null` |
| existingRecord | DailyRecord \| null | `null` |
| allChildren | Child[] | `[]` |
| allRecords | DailyRecord[] | `[]` |
| mood | string \| null | `null` |
| activities | string[] | `[]` |
| selectedPhrases | string[] | `[]` |
| memo | string | `""` |
| aiText | string | `""` |
| arrivalTime | string | `""` |
| departureTime | string | `""` |
| pickupMethod | string | `""` |
| loading | boolean | `true` |
| saving | boolean | `false` |
| generating | boolean | `false` |

**定数:**

```
ACTIVITIES = ["工作", "運動", "学習", "自由遊び", "SST", "おやつ", "外出", "音楽", "調理"]
MOODS = { good: "😊良好", neutral: "😐普通", bad: "😢不調" }
PICKUP_METHODS = ["送迎車", "保護者送迎", "自力通所", "その他"]
```

**UI構成（上から順）:**
1. **ヘッダー:** 戻るボタン、児童アイコン + 名前、編集中バッジ（既存記録時）
2. **来所時間 / 退所時間:** time input 2つ横並び。新規時は arrivalTime に現在時刻を自動入力
3. **気分:** 3つのボタン横並び（good/neutral/bad）。選択中は `border-primary bg-primary-light`
4. **活動:** Chip コンポーネントで9種類をグリッド表示（トグル式）
5. **フレーズ:** カテゴリ別グループ表示
   - 児童の domain_tags に一致するフレーズを優先表示
   - デフォルトフレーズ + カスタムフレーズ両方表示
   - Chip コンポーネント（トグル式、選択時は text を配列に追加）
6. **メモ:** textarea（rows=3）
7. **AI生成ボタン:** "AIで文章を作成 v2.0" — `generating` 中はスピナー表示
8. **AI記録テキスト:** textarea（rows=6）、AI生成後に表示・手動編集可能
9. **送迎方法:** Chip 4つ（排他選択）
10. **保存ボタン:**
    - 新規: "保存して次へ"
    - 編集: "更新する"
    - `saving` 中は "保存中..." 表示

**保存後の自動遷移:**
- 次の未記録児童がいる → その児童の `/record/[nextChildId]` へ遷移
- 全員記録済み → `/` へ遷移

## 4. 共通 UI コンポーネント仕様

### 4.1 Button

| Prop | 型 | デフォルト |
|---|---|---|
| variant | `"primary" \| "secondary" \| "danger" \| "ghost"` | `"primary"` |
| fullWidth | boolean | `false` |
| disabled | boolean | `false` |
| children | ReactNode | — |

**バリアント別スタイル:**
| variant | 背景 | テキスト | ホバー |
|---|---|---|---|
| primary | `bg-primary` | white | `hover:bg-primary/90` |
| secondary | `bg-white border` | `text-foreground` | `hover:bg-gray-50` |
| danger | `bg-danger` | white | `hover:bg-danger/90` |
| ghost | transparent | `text-foreground` | `hover:bg-gray-100` |

共通: `rounded-xl px-4 py-3`、`disabled:opacity-50 disabled:cursor-not-allowed`

### 4.2 Card

| Prop | 型 | デフォルト |
|---|---|---|
| padding | boolean | `true` |
| className | string | `""` |

スタイル: `rounded-2xl border bg-white`、padding 時は `p-4`

### 4.3 Input / Textarea

| Prop | 型 | デフォルト |
|---|---|---|
| label | string | — |
| error | string | — |

- Input: `rounded-xl border px-4 py-3`、`focus:border-primary focus:ring-1`
- Textarea: 同上、`rows=3`
- エラー表示: `text-sm text-red-500 mt-1`

### 4.4 Chip

| Prop | 型 | デフォルト |
|---|---|---|
| label | string | — |
| selected | boolean | `false` |
| onToggle | () => void | — |
| size | `"sm" \| "md"` | `"md"` |

- 選択時: `border-primary bg-primary-light text-primary`
- 非選択時: `border-border bg-white`
- 共通: `rounded-full`、トランジション付き

### 4.5 Header

| Prop | 型 | デフォルト |
|---|---|---|
| title | string | — |
| showBack | boolean | `false` |
| right | ReactNode | — |
